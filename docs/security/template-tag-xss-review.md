## Security Pattern

All reviewed instances follow the same secure pattern:

1. **Use `format_html()` for individual HTML fragments**
   - Django's `format_html()` automatically escapes all user input
   - Treats format string as trusted, parameters as untrusted

2. **Join pre-escaped fragments**
   - Only after `format_html()` has escaped user input
   - `mark_safe()` is used on already-safe HTML

3. **Trust only static content**
   - Bootstrap CSS classes
   - HTML structure and attributes
   - Icon class names

4. **Never pass raw user input to `mark_safe()`**
   - Always escape through `format_html()` first
   - Or use only trusted constants

## Conclusion

- Static content (Bootstrap classes) is correctly trusted
